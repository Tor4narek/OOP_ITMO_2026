# Лабораторная работа №5

## Архитектура приложения, SOLID и Dependency Injection

---

# Цель работы

Освоение проектирования архитектуры приложения на C# с использованием **SOLID**, слоистой архитектуры и **Dependency Injection**.

В рамках работы студент:

* проектирует архитектуру приложения перед реализацией;
* разделяет приложение на архитектурные слои;
* применяет принципы **SOLID**;
* внедряет зависимости через **Dependency Injection**;
* реализует сервисный слой приложения;
* использует **Entity Framework Core** в рамках архитектуры.

# Общая идея приложения

В предыдущих лабораторных работах приложение постепенно усложнялось:

* данные хранились **в памяти**;
* затем **в файлах**;
* затем **в базе данных через Entity Framework Core**.

В данной лабораторной работе необходимо перейти к следующему этапу — **архитектурной организации приложения**.

Основные принципы:

* приложение реализуется как **чистый монолит**;
* код разделяется на **архитектурные слои**;
* зависимости между компонентами управляются через **Dependency Injection**;
* бизнес-логика не зависит от реализации хранения данных;
* приложение остается **консольным**.

# Техническое задание


# 1. Проектирование архитектуры

## Задача

Перед реализацией программы необходимо спроектировать архитектуру приложения.

## Требования

Приложение должно быть реализовано как **чистый монолит** с разделением на проекты.

Рекомендуемая структура solution:

```
Solution
├ Domain
├ Application
├ Infrastructure
└ Presentation.Console
```

Назначение проектов:

### Domain

Содержит:

* сущности предметной области;
* перечисления (`enum`);
* объекты-значения при необходимости.

Слой **не должен зависеть от других проектов**.


### Application

Содержит:

* прикладные сервисы;
* бизнес-логику приложения;
* сценарии работы системы;
* интерфейсы сервисов.


### Infrastructure

Содержит:

* `DbContext`;
* конфигурацию Entity Framework Core;
* миграции;
* реализацию доступа к данным.


### Presentation.Console

Содержит:

* консольное меню;
* взаимодействие с пользователем;
* настройку Dependency Injection;
* точку входа в приложение.


## UML-диаграмма архитектуры

Перед реализацией программы студент должен представить **UML-диаграмму архитектуры приложения**.

Диаграмма должна отражать:

* проекты solution;
* зависимости между проектами;
* основные классы приложения;
* основные интерфейсы;
* сущности предметной области;
* прикладные сервисы;
* `DbContext`;
* связи между компонентами системы.

Диаграмма должна показывать, **какие именно классы и интерфейсы будут реализованы в программе**.

После согласования диаграммы с преподавателем студент может приступать к реализации.


# 2. Применение принципов SOLID

## Задача

Спроектировать код таким образом, чтобы он соответствовал принципам **SOLID**.

В программе должны быть применены следующие принципы.

### Single Responsibility Principle

Каждый класс должен иметь **одну ответственность**.


### Open / Closed Principle

Код должен быть **открыт для расширения**, но **закрыт для модификации**.


### Liskov Substitution Principle

Наследники должны корректно заменять базовые типы.


### Interface Segregation Principle

Интерфейсы должны быть **небольшими и специализированными**.


### Dependency Inversion Principle

Зависимости между компонентами должны строиться **через абстракции**.


# 3. Service Layer

## Задача

Выделить **сервисный слой**, содержащий бизнес-логику приложения.

## Требования

Сервисы должны:

* реализовывать прикладные сценарии работы;
* взаимодействовать с базой данных;
* вызываться из слоя представления.

Пример структуры:

```
Application
 ├ Services
 │   ├ EntityService
 │   └ AnotherEntityService
```

Слой **Presentation** не должен напрямую работать с базой данных.


# 4. Dependency Injection

## Задача

Организовать внедрение зависимостей через контейнер DI.

## Требования

Необходимо использовать библиотеку:

```
Microsoft.Extensions.DependencyInjection
```

Регистрация зависимостей выполняется в проекте `Presentation.Console`.

Пример:

```csharp
var services = new ServiceCollection();

services.AddDbContext<AppDbContext>();

services.AddScoped<EntityService>();

var provider = services.BuildServiceProvider();
```

Все зависимости должны внедряться через **конструкторы классов**.

Создание зависимостей через `new` внутри сервисов **не допускается**.


# 5. Использование Entity Framework Core

## Задача

Использовать базу данных для хранения данных приложения.

## Требования

Работа с базой данных должна выполняться через:

* `DbContext`
* `DbSet<T>`
* `async / await`
* миграции EF Core.

Строка подключения должна храниться в `appsettings.json`.

Пример:

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Server=.;Database=Lab5Db;Trusted_Connection=True;TrustServerCertificate=True;"
  }
}
```


# 6. Реализация функциональности

Приложение должно поддерживать операции:

### Create

создание новой записи.

### Read

получение:

* одной записи
* списка записей.

### Update

изменение существующей записи.

### Delete

удаление записи.

Все операции должны выполняться через **сервисный слой**.
