# Лабораторная работа №1

## Консольное объектно-ориентированное приложение

---

## Цель работы

Освоение основ объектно-ориентированного программирования в языке C#:
работа с **классами, структурами и перечислениями**, понимание различий между **value types и reference types**,
использование **конструкторов, свойств, методов, static и const**,
применение **наследования, абстрактных классов, интерфейсов, полиморфизма и исключений**.

Развитие навыков проектирования архитектуры консольного приложения и реализации предметной логики.

---

## Выбор варианта

Номер варианта определяется по таблице вариантов.
Вариант определяет **предметную область и набор действий**, доступных пользователю.

---

## Общая идея приложения

Студент реализует **консольное приложение**, позволяющее выполнять **основное действие предметной области**
(например, бронирование номера в отеле).

* Все данные хранятся **только в памяти программы**
* Для хранения данных используются **коллекции**
* Приложение работает в рамках **одного запуска**
* Архитектура должна позволять в будущем расширение (следующие лабораторные)

## Техническое задание

---

### **1. Базовые типы данных**

**Задача:**
Определить вспомогательные типы данных предметной области.

#### Требования:

* Реализовать **как минимум одно перечисление (`enum`)**
  (статус, тип, категория, состояние и т.п.).
* Реализовать **как минимум одну структуру (`struct`)**, предназначенную для:

  * хранения группы логически связанных данных
  * передачи по значению
* `struct` должен:

  * иметь конструктор
  * использоваться внутри классов
* В коде должна быть продемонстрирована разница между `struct` и `class`.

---

### **2. Абстрактный уровень архитектуры**

**Задача:**
Определить общие контракты и базовую сущность системы.

#### Требования:

* Реализовать:

  * **как минимум один интерфейс**
  * **один абстрактный класс**
* Интерфейс должен описывать **действие предметной области**
  (например: бронирование, аренда, выдача и т.п.).
* Абстрактный класс должен:

  * представлять обобщённую сущность предметной области
  * содержать общие свойства и методы
  * содержать абстрактный или виртуальный метод
  * иметь один или несколько конструкторов
  * использовать `this`, `static` и `const`

---

### **3. Наследование и конкретные реализации**

**Задача:**
Реализовать конкретные классы, участвующие в логике приложения.

#### Требования:

* Реализовать **не менее двух классов-наследников** абстрактного класса.
* Каждый класс должен:

  * иметь собственный конструктор
  * вызывать `base(...)`
  * переопределять (`override`) методы базового класса
* Поведение классов должно **логически отличаться**.
* Работа с объектами должна выполняться:

  * через базовый класс **или**
  * через интерфейс
    (демонстрация полиморфизма обязательна).

---

### **4. Исключения и обработка ошибок**

**Задача:**
Реализовать корректную обработку ошибок предметной логики.

#### Требования:

* Использовать стандартные исключения:

  * `ArgumentException`
  * `InvalidOperationException` (или аналоги)
* Реализовать **как минимум одно пользовательское исключение**.
* Исключения должны выбрасываться при:

  * некорректном вводе данных
  * невозможности выполнения действия
    (например, попытка забронировать занятый номер)
* В `Main` обязательно использовать `try-catch-finally`.

---

### **5. Консольный сценарий работы**

**Задача:**
Реализовать интерактивное взаимодействие с пользователем через консоль.

#### Требования:

Приложение должно:

* Отображать **простейшее меню** (без бесконечного CRUD):

  * выполнить основное действие
  * вывести текущее состояние данных
  * выход
* Получать ввод пользователя через `Console.ReadLine`
* Выполнять действие предметной области
* Хранить данные в **коллекциях** (`List<T>` или аналог)
* Корректно обрабатывать ошибки ввода и логики

---

### **6. Демонстрация тем лекций**

В программе **обязательно** должно быть использовано:

* `struct`, `enum`
* `class`, `abstract class`
* интерфейсы
* наследование и полиморфизм
* конструкторы (1+)
* `this`, `static`, `const`
* nullable-типы
* исключения (`try-catch`, `throw`)
